## ğŸ¨ How to Change Your App Logo

### Quick Method (Recommended)

1. **Create your new logo:**
   - **Size:** 1024x1024 pixels
   - **Format:** PNG with transparency
   - **Important:** No transparency for iOS (use solid background)
   - **Shape:** Square (iOS will automatically round corners)

2. **Replace the icon file:**
   ```bash
   # Backup the old icon (optional)
   mv assets/icon.png assets/icon-old.png

   # Copy your new icon
   cp /path/to/your/new-icon.png assets/icon.png
   ```

3. **Regenerate iOS assets:**
   ```bash
   cd nora-mobile

   # Clean and rebuild iOS folder
   npx expo prebuild --clean --platform ios
   ```

4. **Done!** Your new icon is ready for the next build.

## ğŸ–¼ï¸ Design Tips

1. **Keep it simple:** Icons are displayed at small sizes (60x60 to 120x120)
2. **Use bold shapes:** Thin lines may not be visible
3. **Avoid text:** Text is usually too small to read
4. **Test on device:** Check how it looks on actual phones
5. **Use high contrast:** Make sure it's visible on all backgrounds

## â“ Troubleshooting

### Icon doesn't update after replacing file

**Solution:**
```bash
# Clean rebuild
cd nora-mobile
npx expo prebuild --clean --platform ios

# If still not working, full clean
rm -rf ios android
npx expo prebuild
```

### Icon looks blurry

**Issue:** Source image is too small or low quality

**Solution:** Use a 1024x1024 PNG with high resolution (at least 72 DPI)

### Icon has white background on Android

**Issue:** Transparency not handled correctly

**Solution:** Create a separate `assets/adaptive-icon.png` with transparent background

### TestFlight shows old icon

**Issue:** Didn't rebuild after changing icon

**Solution:**
1. Update `assets/icon.png`
2. Run `npx expo prebuild --clean --platform ios`
3. Archive new build in Xcode
